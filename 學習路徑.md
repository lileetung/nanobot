## 1. 專案概覽

### 是什麼？

Nanobot 是一個**極簡 AI Agent 框架**，讓你用最少的程式碼把 LLM 接進各種聊天平台。它的設計哲學是：

- **超輕量**：比同類框架少 99% 的程式碼
- **可研究**：程式碼簡潔，適合學習 Agent 架構
- **可生產**：支援 9+ 聊天平台、20+ LLM 供應商

### 核心概念速覽

```
使用者 (Telegram/Discord/等)
    ↓
Channel（訊息接收）
    ↓
Message Bus（訊息路由）
    ↓
Agent Loop（AI 推理 + 工具執行）
    ↓
LLM Provider（Claude/GPT/等）
    ↓
回應傳回使用者
```

### Step 3：追蹤一條訊息的完整生命週期（1 天）

從 `nanobot/channels/telegram.py` 開始，追蹤訊息流：

```
1. Telegram 收到使用者訊息
   → telegram.py 的訊息接收 callback

2. 轉換為 InboundMessage
   → bus/events.py 查看資料結構

3. 放入 Message Bus
   → bus/queue.py 的 put() 方法

4. Agent Loop 取出訊息
   → agent/loop.py 的主循環

5. 建構 LLM Prompt
   → agent/context.py

6. 呼叫 LLM
   → providers/litellm_provider.py

7. 執行工具（如果 LLM 要求）
   → tools/registry.py 分派到各工具

8. 輸出 OutboundMessage
   → 返回 Bus → 回到 Telegram
```

**實作練習**：用 Python debugger 或加 `print()` 追蹤一次完整流程

### Module 1：Agent Loop（最重要）

**檔案**：`nanobot/agent/loop.py`（約 510 行）

**關鍵概念**：

```python
# 理解這個核心循環
while True:
    message = await bus.get()           # 等待訊息
    context = build_context(message)    # 建構 Prompt
    response = await llm.call(context)  # 呼叫 LLM

    while response.has_tool_calls():    # Tool calling 循環
        results = await execute_tools(response.tool_calls)
        response = await llm.call(context + results)

    await bus.send(response)            # 回應使用者
```

### MCP（Model Context Protocol）整合

MCP 讓 Nanobot 連接外部工具服務：

```json
// 在 config.json 中配置 MCP server
{
  "tools": {
    "mcp_servers": [
      {
        "name": "my_mcp",
        "command": "npx",
        "args": ["-y", "@some/mcp-server"]
      }
    ]
  }
}
```

**學習重點**：
- [ ] `tools/mcp.py` 如何與 MCP server 通訊？
- [ ] MCP 工具如何整合到 Agent 的 tool_use 流程？

### 多 Agent 架構
