⏺ 建議的學習路徑

  第一階段：理解資料流（1-2 小時）

  從一條訊息的完整生命週期開始，依序讀這 5 個檔案：

  ✅ 1. nanobot/bus/events.py — 只有 37 行，定義 InboundMessage 和 OutboundMessage，這是整個系統的「血液」
  ✅ 2. nanobot/channels/base.py — 60 行，理解頻道抽象：start() 監聽、send() 回覆
  ✅ 3. nanobot/channels/slack.py — 挑一個具體頻道看怎麼實作 base
  ✅ 4. nanobot/bus/queue.py — 事件匯流排怎麼把訊息從頻道送到 Agent
  ✅ 5. nanobot/agent/loop.py — 最核心的檔案，Agent 迴圈的 5 步驟：收訊息 → 組上下文 → 呼叫 LLM → 執行工具 → 回應

  讀完這條線，你就掌握了 80% 的架構。

  第二階段：深入 Agent 內部機制（2-3 小時）

  ✅ 6. nanobot/agent/context.py — 提示詞是怎麼組裝的（SOUL + TOOLS + 記憶 + 歷史）
  7. nanobot/agent/memory.py — 記憶的讀寫與整合策略
  8. nanobot/agent/tools/base.py + registry.py — 工具系統的抽象，每個 tool 怎麼註冊
  9. nanobot/templates/SOUL.md — Agent 的人格設定，理解 prompt engineering 的實踐

  第三階段：設定與 Provider 層（1 小時）

  10. nanobot/providers/base.py — LLM 抽象介面（LLMResponse 支援 tool_calls、reasoning、thinking）
  11. nanobot/providers/litellm_provider.py — 最通用的實作，支援多模型
  12. nanobot/config/schema.py → loader.py — 整個系統怎麼被設定驅動

  第四階段：進階功能（按興趣）

  13. nanobot/heartbeat/ — Agent 自主思考的定時觸發機制
  14. nanobot/cron/ — 排程系統
  15. nanobot/skills/ — 插件化技能，看 SKILL.md 的格式就能理解擴充方式
  16. nanobot/agent/subagent.py — 子代理人的生成與管理
