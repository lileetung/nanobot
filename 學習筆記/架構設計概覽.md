Nanobot 是一個多平台 AI Agent 框架，核心設計圍繞幾個關鍵層次：

┌──────────┬───────────────────────┬──────────────────────────────────────────┐
│   層次    │         模組          │                   職責                    │
├──────────┼───────────────────────┼──────────────────────────────────────────┤
│ 入口     │ cli/, __main__.py     │ CLI 啟動、設定載入                          │
├──────────┼───────────────────────┼──────────────────────────────────────────┤
│ 頻道層    │ channels/             │ 統一 10+ 通訊平台的訊息收發                 │
├──────────┼───────────────────────┼──────────────────────────────────────────┤
│ Agent 層 │ agent/                │ 對話迴圈、上下文管理、記憶、工具呼叫           │
├──────────┼───────────────────────┼──────────────────────────────────────────┤
│ 供應商層  │ providers/            │ 抽象 LLM API（LiteLLM、Azure、Codex 等）   │
├──────────┼───────────────────────┼──────────────────────────────────────────┤
│ 事件層    │ bus/                  │ 模組間解耦的事件驅動通訊                     │
├──────────┼───────────────────────┼──────────────────────────────────────────┤
│ 擴充層    │ skills/, agent/tools/ │ 插件式技能與工具擴充                        │
├──────────┼───────────────────────┼──────────────────────────────────────────┤
│ 排程層    │ cron/, heartbeat/     │ 定時任務與自主心跳                          │
└──────────┴───────────────────────┴──────────────────────────────────────────┘

核心流程：頻道收到訊息 → 事件匯流排分發 → Session 管理 → Agent 迴圈（思考/工具/回應）→ 頻道回覆。heartbeat 則讓 Agent 能在沒有外部觸發時自主思考。
